你要为 Astro Paper 博客生成一篇中文《AI 工程日报》。

任务：{task}
日期：{date}

以下候选源已经经过全局分类，只属于 `ai-daily`。只能使用这些候选源里的事实和链接：

{source_text}

写作定位：
- 这是 AI 工程日报，不是 AI 新闻垃圾桶。
- 关注模型、Agent、AI infra、AI 编程、评测、安全治理、RAG/向量数据库、企业 AI 落地中的权限/评测/成本/工程问题。
- 不写普通融资、工具榜单、纯 prompt 技巧、低信号论文搬运、纯公司商业新闻。

# 输出要求

只输出一个合法 JSON 对象，不要输出 Markdown、frontmatter、代码围栏或任何解释。

链接由脚本从候选源的 `- 链接：` 字段按 `source_url` 校验回填，你只需把选定的来源链接放进 `source_url`，
不要自己拼 Markdown 链接语法。archive 层会据此生成各栏目 + `### [标题](链接)` 的最终文章。

JSON 结构必须严格如下：

```json
{
  "sections": [
    {
      "title": "中文栏目名",
      "items": [
        {
          "title_zh": "中文事件标题",
          "source_url": "来自候选源 - 链接： 字段的原始链接",
          "body_markdown": "150-300 字正文段落"
        }
      ]
    }
  ]
}
```

内容要求：
- 按候选源顺序逐条写；候选源有多少条，最多写多少条；不要新增候选源以外的条目。
- 每个候选条目最多使用一次。
- `title_zh` 必须翻译或改写为自然中文标题，保留模型名、产品名、公司名等专有名词，不要直接照搬英文原题。
- `source_url` 必须逐字使用候选源 `- 链接：` 字段。
- 每条 `body_markdown` 150～300 字，必须包含“变化点 + AI 工程影响 + 风险/边界/适用对象”。
- 如果候选很少，也照常写已有高质量条目，不要凑数。

栏目建议：
- `今日模型与产品`
- `Agent 与工程化`
- `AI Infra、成本与数据`
- `安全、评测与治理`

过滤规则：
- 禁止：`融资`、`工具榜单`、`提示词技巧`、`论文导读`。
- 禁止教程腔：`一文搞懂`、`一文读懂`、`从零`、`从零开始`、`入门教程`。
- 禁止空话：`赋能`、`颠覆`、`革命性`、`不容错过`、`值得关注`。
- 不要写“本文将”“下面我们来”“总之”。
