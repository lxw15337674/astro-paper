# 输出要求

只输出可归档的 Markdown 正文，不要输出 frontmatter，不要输出代码围栏。

文章必须按以下二级标题顺序输出：

- `## 总结`
- `## 美股`
- `## A股`
- `## 港股`
- `## BTC 市场动态`

# 总结规则

- `## 总结` 必须放在正文最上面。
- 总结只概括当天可复核市场状态、休市状态和数据边界。
- 如果某个市场当日未产生完整常规交易数据，必须在总结里说明，不要用最近一个交易日假装当天开盘。

# 未开盘/缺数据固定文本

如果输入证据中某个市场已经给出以下固定文本，输出时必须原样保留，不能改写成行情判断：

- 美股：`美股当日未产生完整常规交易数据，本节不做涨跌与板块强弱判断。`
- A股：`A股当日未产生完整常规交易数据，本节不做指数表现与市场结构判断。`
- 港股：`港股当日未产生完整常规交易数据，本节不做指数表现与市场结构判断。`

如果输入证据写的是“未获取到完整常规交易数据”，也必须保留该数据边界，不要补行情。

# 写作风格

- 中文、清晰、克制，像给投资者和工程读者看的自动化市场简报。
- 优先解释“数据说明什么”和“哪些不能从当前数据推出”。
- 不要写鸡汤、标题党或无法由证据支持的交易建议。
- 禁止出现“建议关注”“值得关注”“继续关注”“最看好”“赚钱点子”“操作”“布局”“机会”等动作导向语言。
- 不要出现“待补充”“示例”“稍后更新”等占位文字。
