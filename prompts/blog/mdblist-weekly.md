# 任务

你要把下面的影视榜单候选证据，写成一篇中文《每周影视推荐》博客正文。证据字段大多是英文，你需要翻译并改写成自然中文。

- 任务：{task}
- 日期：{date}

# 输入证据

```md
{source_text}
```

# 输出结构

正文分为两个大类，各自用二级标题：

- `## 电影推荐`
- `## 剧集推荐`

把「电影候选」写进 `## 电影推荐`，「剧集候选」写进 `## 剧集推荐`。每个候选作品写成一个小节，顺序沿用证据顺序：

```
### 中文译名（English Title）

![中文译名 海报](证据里的海报链接)

#### 基本信息

- 类型：把「题材(EN)」译成中文，用、分隔
- 上映日期：把「上映日期」写成中文（例：2026-05-13 → 2026年5月13日；只有年份就写该年）
- IMDb 评分：取「评分」行里的 IMDb 数值

#### 剧情概要

把「简介(EN)」翻译并改写成一段自然中文，抛出看点或悬念，但不要剧透结局。

#### 推荐理由

基于题材、评分、剧情走向和整体气质做分析，说清它适合什么观众、看点在哪。

#### 评论总结

基于「评分」行里的真实多平台评分（IMDb / 烂番茄 / Trakt）概括口碑——评分是否一致、偏正面还是有分歧、在同类里的定位。
```

# 写作要求

- 标题：把英文原名翻成通行的中文译名，括号里保留英文原名；已有官方中文名就用官方名。
- 海报：必须原样使用证据里「海报」字段给出的链接；证据里海报是 `-` 时，省略该图片行，不要编造链接。
- 语气参考影视编辑栏目：有判断、有观点，不是干巴巴罗列字段。
- 每个作品的「剧情概要 + 推荐理由 + 评论总结」合计约 200–400 字。

# 质量边界

- 不要点名导演或演员——证据里没有主创信息，编造属于严重错误。
- 不要编造评分、剧情、上映日期，也不要虚构「有观众说……」这类评价原话；评论总结只能基于给出的评分数据。
- 不要把英文字段原样堆进正文，要翻译改写。
- 不要出现「待补充」「示例」「信息不足」「无法判断」等占位文字。
- 只输出可归档的 Markdown 正文，不要输出 frontmatter，不要输出代码围栏。
