# 任务

你要把下面的影视榜单候选证据，翻译并改写成一篇中文《每周影视推荐》。证据字段大多是英文，你需要翻译并改写成自然中文。

- 任务：{task}
- 日期：{date}

# 输入证据

```md
{source_text}
```

# 输出要求

只输出一个合法 JSON 对象，不要输出 Markdown、frontmatter、代码围栏或任何解释。

原标题、海报链接、IMDb 评分、上映日期由脚本从证据回填，你不要输出它们。你只需为每部作品产出
「中文译名、类型中文、剧情概要、推荐理由、评论总结」这几个语义字段。archive 层会用 `movies`/`series`
两个数组分别对应证据里的「电影候选」「剧集候选」，按 `rank` 与证据顺序合并，生成最终博客文章。

JSON 结构必须严格如下（`rank` 与各自证据列表里的序号一一对应）：

```json
{
  "movies": [
    {
      "rank": 1,
      "title_zh": "中文译名",
      "genres_zh": "把题材译成中文，用、分隔",
      "plot": "把简介翻译并改写成一段自然中文，抛出看点或悬念，不剧透结局",
      "recommendation": "基于题材、评分、剧情走向和整体气质，说清适合什么观众、看点在哪",
      "review": "基于证据里的多平台评分概括口碑：是否一致、偏正面还是有分歧、在同类里的定位"
    }
  ],
  "series": [
    { "rank": 1, "title_zh": "…", "genres_zh": "…", "plot": "…", "recommendation": "…", "review": "…" }
  ]
}
```

# 写作要求

- `title_zh`：把英文原名翻成通行的中文译名；已有官方中文名就用官方名。原英文名由脚本回填在括号里，你不用写。
- 语气参考影视编辑栏目：有判断、有观点，不是干巴巴罗列字段。
- 每部作品的「plot + recommendation + review」合计约 200–400 字。
- 不要在 JSON 里输出海报链接、IMDb 评分、上映日期；它们由脚本回填。

# 质量边界

- 不要点名导演或演员——证据里没有主创信息，编造属于严重错误。
- 不要编造评分、剧情、上映日期，也不要虚构「有观众说……」这类评价原话；`review` 只能基于给出的评分数据。
- 不要把英文字段原样堆进正文，要翻译改写。
- 不要出现「待补充」「示例」「信息不足」「无法判断」等占位文字。
