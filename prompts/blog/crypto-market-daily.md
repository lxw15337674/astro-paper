# 任务

你要为 Astro Paper 博客生成一篇中文数字货币日报正文。

- 任务：{task}
- 日期：{date}

# 输入证据

下面是脚本抓取的数字货币全市场、主流资产、分类板块与数据边界。只能依据这些证据写作；不要补不存在的链上数据、交易所数据、政策或宏观解释。

```md
{source_text}
```

# 输出要求

只输出可归档的 Markdown 正文，不要输出 frontmatter，不要输出代码围栏。

文章必须包含并优先保持以下结构：

- `## 总结`
- `## 全市场概览`
- `## 主流资产表现`
- `## 市场强弱结构`
- `## 数据边界`

# 总结规则

- `## 总结` 必须放在正文最上面。
- 总结必须覆盖全市场总市值、24小时成交量、BTC/ETH 占比、主流资产表现或缺失边界。
- 不能只写 BTC，也不能把 BTC 单独当作整个数字货币市场。

# 数据边界

- 分类板块、涨跌排行和市值过滤必须按输入证据描述。
- 如果输入证据写明未获取到完整数据，必须保留数据边界，不要补行情。
- 不能把生成时点的 24 小时变化伪装成交易所统一收盘价。

# 写作风格

- 中文、清晰、克制，像给投资者和工程读者看的自动化市场简报。
- 优先解释“数据说明什么”和“哪些不能从当前数据推出”。
- 不要写鸡汤、标题党或无法由证据支持的交易建议。
- 禁止出现“建议关注”“值得关注”“继续关注”“后续关注”“最看好”“赚钱点子”“操作”“布局”“机会”“交易建议”“投资建议”等动作导向语言。
- 不要出现“待补充”“示例”“稍后更新”等占位文字。
