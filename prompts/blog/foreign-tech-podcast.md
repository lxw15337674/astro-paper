# 输出要求

只输出可归档的 Markdown 正文，不要输出 frontmatter，不要输出代码围栏。

文章结构必须包含：

1. `《今日国外热门科技访谈播客》`
2. `## 今日总览`
3. `## 今日播客清单`
4. 每期播客一个 `## episode title` 小节
5. 每个 episode 小节里保留基本信息：节目、嘉宾、日期、来源、链接

# 质量要求

- 必须是长文中文笔记，不是短消息摘要。
- 重点写访谈里的技术、产品、职业、产业判断和可复盘观点。
- 不要编造嘉宾、链接、来源或节目名。
- 不要出现“待补充”“示例”“信息不足”等占位文字。
