# 任务

根据一篇《{magazine}》文章的完整 EPUB 正文生成中文文章级 JSON。你是本期导读的上游步骤，你的结果会被程序固定使用，后续不会再被改写。

- 刊期归档日期：{date}
- 文章编号：{rank}

# 输入证据

```md
{article_text}
```

# 输出格式

只输出一个合法 JSON 对象，不要输出 Markdown、代码围栏或任何解释。必须严格使用输入文章编号：

```json
{
  "rank": {rank},
  "title_zh": "自然、准确的中文标题",
  "one_sentence_summary": "一句话说明文章讨论什么和结论是什么。",
  "core_point": "明确核心论点、关键因果或分歧。",
  "content_summary": "Markdown 正文：完整交代背景、主要事实、论证过程、结论和必要边界，不复刻正文。"
}
```

- `content_summary` 使用 Markdown 正文格式：可用多个自然段、`-` 或 `1.` 列表、`**加粗**` 与 `>` 引用，用它们让长内容更有条理、可读。
- `content_summary` 禁止使用任何标题语法（`#`、`##`、`###`、`####` 等）；标题层级由后续程序统一控制。
- 其余字段（`title_zh`、`one_sentence_summary`、`core_point`）保持单行纯文本，不要使用 Markdown 结构。

不限制各字段字数，但内容必须完整、具体，不能用不同说法重复同一段内容。`title_zh` 必须使用中文，不要附带英文原题。不得编造引语、数字、立场、链接或输入中不存在的事实；不得提及 EPUB、模型或提示词。
