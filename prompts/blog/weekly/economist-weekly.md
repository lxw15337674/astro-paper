# 输出格式

只输出一个合法 JSON 对象，不要输出 Markdown、frontmatter、代码围栏或解释。

每篇文章的中文标题与三类摘要已经由独立请求生成并固定，不能改写、缩写、扩写、排序、遗漏或重复。你只负责生成整期简介、主题脉络与阅读路线，不要返回文章数组。

```json
{
  "description": "本期导读摘要",
  "issue_overview": "梳理本期所有文章共同呈现的主题、分歧与背景；只基于输入证据。",
  "reading_route": "用一段中文给出阅读优先级与路线：时间有限先读哪些文章、为何如此安排。"
}
```

# 质量边界

- 只能根据证据改写，不能编造作者、事实、引语、数据、文章立场或原文链接。
- 不要逐篇重述文章摘要，不要输出文章数组或任何单篇字段。
- 不要逐句翻译、不要输出长引文，也不要提及 EPUB、抓取、模型或输入证据。
- 必须综合全部文章，避免“值得关注”“内容丰富”等空泛表述。
