# 任务

把下方《经济学人》本期文章级证据整理为一篇中文综合导读。

- 任务：{task}
- 归档日期：{date}

# 输入证据

```md
{source_text}
```

# 输出格式

只输出一个合法 JSON 对象，不要输出 Markdown、frontmatter、代码围栏或解释。

原标题、栏目、作者、原文链接、刊期由脚本从证据回填；不得输出或改写这些事实字段。必须严格覆盖证据中每一个编号文章，按 `rank` 一一对应，不得遗漏、重复、调换或自行增加文章。目标是让读者即使不打开原文，也能准确知道文章讲了什么。

```json
{
  "description": "本期导读摘要，40字以内",
  "issue_overview": "2—4 段中文，梳理本期多篇文章共同呈现的主题、分歧与背景；只基于输入证据。",
  "articles": [
    {
      "rank": 1,
      "title_zh": "自然、准确的中文标题",
      "one_sentence_summary": "40—80 字，一句话说明文章在讨论什么、得出什么判断。",
      "core_point": "100—180 字，明确文章的核心论点、关键因果或分歧。",
      "content_summary": "220—400 字，完整交代背景、主要事实、论证过程、结论和必要边界，不复刻正文。"
    }
  ],
  "reading_route": "用一段中文给出阅读优先级与路线：时间有限先读哪些文章、为何如此安排。"
}
```

# 质量边界

- 只能根据证据改写，不能编造作者、事实、引语、数据、文章立场或原文链接。
- `title_zh` 必须是中文；不要在其中附原英文标题。
- 不要逐句翻译、不要输出长引文，也不要提及 EPUB、抓取、模型或输入证据。
- 三个字段各自承担不同职责，禁止换一种说法重复同一段内容；避免“值得关注”“内容丰富”等空泛表述。
