# 任务

你要把下面的纽约时报畅销书榜候选证据，翻译并改写成一篇中文《每周图书推荐》。证据字段大多是英文，你需要翻译并改写成自然中文。

- 任务：{task}
- 日期：{date}

# 输入证据

```md
{source_text}
```

# 输出要求

只输出一个合法 JSON 对象，不要输出 Markdown、frontmatter、代码围栏或任何解释。

原书名、作者、出版社、榜单排名、上榜周数、封面链接、书评链接由脚本从证据回填，你不要输出它们。你只需为每本书产出
「中文书名、类型、内容简介、推荐理由」这几个语义字段。archive 层会用 `fiction`/`nonfiction`
两个数组分别对应证据里的「小说候选」「非虚构候选」，按 `rank` 与证据顺序合并，生成最终博客文章。

JSON 结构必须严格如下（`rank` 与各自证据列表里的序号一一对应）：

- 输入中实际提供的小说和非虚构候选必须全部输出，每个 `rank` 恰好出现一次，不得漏项、重复或自行增删候选；某一类可能只有一两本甚至没有，不要自行补足。
- 若证据里某一类候选段落缺席，对应数组输出为空数组 `[]`。

```json
{
  "description": "本周推荐图书的类型与看点，40字以内",
  "fiction": [
    {
      "rank": 1,
      "title_zh": "中文书名",
      "genre_zh": "根据简介判断并用中文标注类型，如长篇小说、悬疑惊悚、爱情、科幻等，用、分隔",
      "summary": "把英文简介翻译并改写成一段自然中文，交代题材与看点，不剧透关键结局",
      "recommendation": "基于题材与故事气质，说清这本书适合什么读者、值得读的地方"
    }
  ],
  "nonfiction": [
    { "rank": 1, "title_zh": "…", "genre_zh": "如回忆录、历史、社科、商业等", "summary": "…", "recommendation": "…" }
  ]
}
```

# 写作要求

- `title_zh`：把英文书名翻成通行的中文译名；已有官方中文版书名就用官方名。原英文书名由脚本回填在括号里，你不用写。
- 语气参考图书编辑荐书栏目：有判断、有观点，不是干巴巴罗列字段。
- 每本书的「summary + recommendation」合计约 150–300 字。
- 不要在 JSON 里输出作者、出版社、排名、封面或书评链接；它们由脚本回填。

# 质量边界

- 不要编造证据里没有的作者、情节、获奖或销量数据。
- `genre_zh` 依据简介合理归类即可，不确定就给宽泛类型，不要杜撰具体子类。
- 不要把英文字段原样堆进正文，要翻译改写。
- 不要出现「待补充」「示例」「信息不足」「无法判断」等占位文字。
