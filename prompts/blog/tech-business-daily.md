你要为 Astro Paper 博客生成一篇中文《科技商业观察日报》。

任务：{task}
日期：{date}

以下候选源已经经过全局分类，只属于 `tech-business-daily`。只能使用这些候选源里的事实和链接：

{source_text}

写作定位：
- 这是科技产业与商业观察日报，不是泛新闻搬运。
- 关注科技公司战略、平台政策、监管/反垄断、芯片/云供应链、重大安全事件的商业影响、企业采购、产业竞争。
- 不写纯技术版本发布、纯 AI 工程实现、消费硬件体验、娱乐、普通融资、营销稿。

# 输出要求

只输出一个合法 JSON 对象，不要输出 Markdown、frontmatter、代码围栏或任何解释。

链接由脚本从候选源的 `- 链接：` 字段按 `source_url` 校验回填，你只需把选定的来源链接放进 `source_url`，
不要自己拼 Markdown 链接语法。archive 层会据此生成各栏目 + `### [标题](链接)` 的最终文章。

JSON 结构必须严格如下：

```json
{
  "sections": [
    {
      "title": "中文栏目名",
      "items": [
        {
          "title_zh": "中文事件标题",
          "source_url": "来自候选源 - 链接： 字段的原始链接",
          "body_markdown": "150-300 字正文段落"
        }
      ]
    }
  ]
}
```

内容要求：
- 按候选源顺序逐条写；候选源有多少条，最多写多少条；不要新增候选源以外的条目。
- 每个候选条目最多使用一次。
- `title_zh` 必须翻译或改写为自然中文标题，保留公司名、平台名、产品名等专有名词，不要直接照搬英文原题。
- `source_url` 必须逐字使用候选源 `- 链接：` 字段。
- 每条 `body_markdown` 150～300 字，必须包含“事件变化 + 影响对象 + 风险/不确定性/后续观察点”。
- 如果候选很少，也照常写已有高质量条目，不要凑数。

栏目建议：
- `今日大事件`
- `公司与平台`
- `政策、监管与安全`
- `市场与供应链信号`

过滤规则：
- 禁止娱乐八卦、购物推荐、工具榜单、融资快讯、投资建议、买卖建议、股价预测。
- 禁止空话：`赋能`、`颠覆`、`革命性`、`不容错过`、`值得关注`。
- 不要写“本文将”“下面我们来”“总之”。
