你要把 Apple Podcasts Top Shows 热门播客证据写成中文长文博客笔记。

- 任务：{task}
- 日期：{date}

# 输入证据

```md
{source_text}
```

# 输出要求

只输出可归档的 Markdown 正文，不要输出 frontmatter，不要输出代码围栏。

文章结构要求：

1. 第一行固定输出：`《今日 Apple Top Shows 热门播客》`
2. 第一行之后直接进入每期播客的 `## 中文播客标题` 小节。
3. 不要输出 `## 今日总览` 或 `## 今日播客清单`，也不要在正文开头额外写总览段、榜单说明或节目清单。
4. 每个 episode 小节必须按以下顺序包含：
   - `### 中文主题`
   - `### 基本信息`
     - `- **节目**：...`
     - `- **嘉宾**：...`，输入没有明确嘉宾时写“未标明”，不要猜。
     - `- **日期**：...`
     - `- **来源**：...`
     - `- **链接**：...`
   - `### 一句话总结`
   - `### Highlights`
     - 3-5 条 bullet，每条必须是可由 transcript 支持的观点、判断、事实机制或社会/商业/文化观察。
   - `### 长文笔记`
     - 至少 3 段，每段围绕一个可复盘观点展开；每段都要提炼知识、观点、依据或启发，不能只描述“聊了什么”。

不要改变上述结构、标题层级、播客条目顺序或整体版式；不要新增引言、榜单点评、节目推荐语、读者可以带走什么等额外大结构。优化重点只在每个播客条目内部，把内容讲解写得更厚。

# 标题翻译要求

- 每期播客的 `##` 小节标题必须翻译成自然中文，不能直接使用英文原题作为标题。
- 标题要忠实表达该期主题，不要标题党化，不要扩写成额外观点。
- 专有名词、产品名、公司名、人名、节目名、机构名可以保留英文，例如 The Daily、Joe Rogan、New York Times、NBA。
- 原始节目名、来源和链接继续放在 `### 基本信息` 内；不要因为标题翻译而改动元数据。

# 质量要求

- 必须是长文中文笔记，不是短消息摘要；每个播客条目都要写到“不听播客也能理解本期具体内容”的程度。
- 长文笔记的目标是把 transcript 提炼成知识和观点，而不是单纯描述播客流程。优先写“这个观点为什么重要、背后的机制是什么、哪些例子支撑它、它对读者判断有什么帮助”。
- Apple Top Shows 只是发现源和排序依据，不是内容立场。不要把“榜单靠前”写成质量背书，也不要写成推荐购买、推荐订阅或价值判断。
- 每个播客条目的 `### 长文笔记` 应尽量讲清：本期主要问题、背景、核心观点、观点理由、例子/事件/人物/公司/制度背景、争议/风险/不确定性、对普通读者理解社会、商业、文化或公共议题的启发。
- `### 长文笔记` 中每段都必须至少承担一种功能：解释一个机制、提炼一个判断、连接一个因果链、比较两种选择、指出一个风险边界、给出一个可迁移启发。只写“节目提到/主持人讨论/他们认为”而没有进一步分析的段落不合格。
- 每个播客条目必须主要依据该条的 `#### Transcript` 写作；榜单排名、标题、链接、图片、嘉宾、show notes 只能做元数据补充，不能单独支撑观点展开。
- 凡是出现一个观点，都要解释它具体指什么、为什么这么说、有什么例子或影响；禁止只写“本期讨论了社会、商业、文化趋势”等泛泛概括。
- 不要按时间线复述播客，也不要把 transcript 压缩成议题列表；应围绕 3-6 个高价值观点重组内容，让读者读完能获得可复用的判断框架。
- 如果某一期播客最终只能让读者知道“大概主题”，但不能知道“里面具体讲了哪些观点、理由、机制、边界和启发”，则视为不合格，必须继续展开该条目。
- 可以做分析和归纳，但只能基于输入证据；不要编造 transcript、嘉宾、链接、来源、节目名、发布日期、榜单排名或未提供的事实。
- 输入源阶段应已过滤掉无 transcript 条目；如果仍看到某条 `#### Transcript` 为空、缺失或写着未提供 transcript，不要为该条生成小节。
- 不要出现“待补充”“示例”“信息不足”“无法判断”“本文将”等占位或元叙述文字；不要向读者解释 source、transcript、证据边界、全文/片段范围或写作过程。
- 不要重复输出同一播客的总结内容；`### 一句话总结`、`### Highlights` 和 `### 长文笔记` 必须各自承担不同功能，不能复制粘贴同一句话或同一段观点。
- 不要生成投资建议、医疗建议、法律建议、产品购买建议或夸张标题。
