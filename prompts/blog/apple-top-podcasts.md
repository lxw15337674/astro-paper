你要把 Apple Podcasts Top Shows 热门播客证据写成中文长文博客笔记。

- 任务：{task}
- 日期：{date}

# 输入证据

```md
{source_text}
```

# 输出要求

只输出可归档的 Markdown 正文，不要输出 frontmatter，不要输出代码围栏。

文章结构要求：

1. 第一行固定输出：`《今日 Apple Top Shows 热门播客》`
2. 第一行之后直接进入每期播客的 `## 中文播客标题` 小节。
3. 不要输出 `## 今日总览` 或 `## 今日播客清单`，也不要在正文开头额外写总览段、榜单说明或节目清单。
4. 每个 episode 小节必须按以下顺序包含：
   - `### 中文主题`
   - `### 基本信息`
     - `- **节目**：...`
     - `- **嘉宾**：...`，输入没有明确嘉宾时写“未标明”，不要猜。
     - `- **日期**：...`
     - `- **来源**：...`
     - `- **链接**：...`
   - `### 内容总结`
     - 2-4 段，按节目真实内容讲清楚这一期到底讲了什么。
   - `### 详细内容`
     - 至少 4 段，按节目推进顺序展开主要人物、事件、材料、对话、分歧、更新和结尾信息。

不要输出 `### 一句话总结`，不要输出 `### Highlights`，不要输出 `### 长文笔记`。不要改变上述结构、标题层级、播客条目顺序或整体版式；不要新增引言、榜单点评、节目推荐语、读者可以带走什么等额外大结构。

# 标题翻译要求

- 每期播客的 `##` 小节标题必须翻译成自然中文，不能直接使用英文原题作为标题。
- 标题要忠实表达该期主题，不要标题党化，不要扩写成额外观点。
- 专有名词、产品名、公司名、人名、节目名、机构名可以保留英文，例如 The Daily、Joe Rogan、New York Times、NBA。
- 原始节目名、来源和链接继续放在 `### 基本信息` 内；不要因为标题翻译而改动元数据。

# 写作重点

- 目标是让没听过播客的读者读完后，清楚知道这一期节目具体讲了什么，而不是只知道“它有什么社会意义”。
- 文章首先是节目内容笔记，不是评论文章、议论文、趋势观察、节目推荐或榜单介绍。
- `### 内容总结` 要复述本期主线：开场问题是什么，节目按什么顺序推进，讲了哪些人物/事件/事实材料，最后留下什么更新、悬念或结论。
- `### 详细内容` 要按照 transcript 的真实推进顺序展开；可以适度解释背景和因果，但解释必须服务于“讲清节目内容”。
- 每一段都必须包含至少一个来自节目证据的具体信息：人物、机构、时间点、案件/事件、节目中的说法、材料、平台动作、数字、地点、产品、公司或制度背景。
- 如果节目是叙事型、案件型或访谈型，要把叙事链条写清楚：谁出现了、他说了什么、为什么这一步影响后续、节目如何转到下一部分。
- 如果节目中有主持人与嘉宾对话，要说明对话双方围绕什么问题交换了哪些观点；不要只抽象成“他们讨论了社会、商业、文化趋势”。
- 可以在段落中解释某个事实为什么重要，但不要让“启发、意义、可迁移框架、对读者而言”压过节目本身。
- 最多只在 `### 详细内容` 末尾用一小段收束这期的重点；不要每段都写成评论式结论。

# 证据与边界

- 每个播客条目必须主要依据该条的 `#### Transcript` 写作；榜单排名、标题、链接、图片、嘉宾、show notes 只能做元数据补充，不能单独支撑正文展开。
- 凡是出现具体事实、人物关系、事件进展或观点归纳，都要能从 transcript 或元数据中找到依据。
- 不要编造 transcript、嘉宾、链接、来源、节目名、发布日期、榜单排名或未提供的事实。
- 输入源阶段应已过滤掉无 transcript 条目；如果仍看到某条 `#### Transcript` 为空、缺失或写着未提供 transcript，不要为该条生成小节。
- 不要出现“待补充”“示例”“信息不足”“无法判断”“本文将”等占位或元叙述文字；不要向读者解释 source、transcript、证据边界、全文/片段范围或写作过程。
- 不要生成投资建议、医疗建议、法律建议、产品购买建议或夸张标题。

# 禁止的旧写法

- 不要把 `### 内容总结` 写成一句话；它必须是能概括节目主线的多段内容摘要。
- 不要写 `### Highlights` bullet 列表。
- 不要把正文主要写成“这期最有价值的是……”“这提醒读者……”“它的可迁移意义在于……”。
- 不要把节目内容拆成抽象观点清单后重组；Apple 热门播客优先保留节目原本的叙事顺序和对话推进。
- 不要让读者读完仍不知道节目里具体发生了什么、谁说了什么、节目从哪里讲到哪里。
