# 输出要求

只输出一个合法 JSON 对象，不要输出 Markdown 代码围栏、文章标题或任何额外解释。

JSON 结构必须严格如下：

```json
{
  "description": "今日市场整体走势的一句话概括，40字以内",
  "overview": "今日总览的完整正文",
  "us": "美股章节的完整正文",
  "ashare": "A股章节的完整正文",
  "hk": "港股章节的完整正文",
  "crypto": "比特币章节的完整正文"
}
```

规则层会依次添加 `## 今日总览`、`## 美股`、`## A股`、`## 港股`、`## 比特币` 标题。字段中只写自然段，不要自行添加任何层级的标题、列表或表格。

# 写作要求

- `overview` 串联美股、A股、港股、BTC，以及证据中可用的债券、汇率、黄金和原油，提炼当天最重要的跨市场主线。
- `us`、`ashare`、`hk`、`crypto` 每节先交代行情与结构，再解释这些结构能说明什么以及不能说明什么。
- 自主选择最有解释力的数据，不要按 JSON 字段顺序逐项复述，也不要为了完整而堆数字。
- 可以使用证据中的精确数字；不能自行换算、推算或引入证据中不存在的数字。需要数字时沿用证据里的精度。
- 数字并非必需：若不能逐字确认数字来自对应字段，请使用定性表述。不要把日期、指数数量或常见阈值当作可自行补充的数字。
- 严格遵守每个市场的 `direction`、`strongest_index` 和 `weakest_index`。`up` 表示已获取的主要指数整体上涨且没有明显下跌，`down` 表示整体下跌且没有明显上涨，`mixed` 表示涨跌分化，`flat` 表示整体接近平盘。
- 当某个市场的 `status` 不是 `open` 时，只说明休市或数据不可用，不补行情、不作强弱推断。

# 事实边界

- 只使用输入证据，不补充政策、宏观事件、公司新闻、链上数据或其它外部知识。
- ETF 涨跌和成交量只能描述风格表现与交易活跃度，不能写成真实资金流入或流出。
- 指数强弱不等于所有成分股或细分行业同步变化。
- Deribit 数据只代表该交易所公开口径，不能写成全市场仓位或交易动作。
- Put/Call、资金费率和 IV 要翻译成普通人能理解的含义，但不能单独推出崩盘、反转或确定性方向。
- 不写投资建议、买卖建议、操作、布局、机会、推荐或喊单语言。
- 使用自然、克制的中文市场编辑语气，避免“值得关注”“持续关注”“数据显示”等模板化填充。
